# 自媒体内容撰写 Skill

## Overview

服务于「易点设计」UED 团队的自媒体内容矩阵。
核心 Slogan：Designing for Business Impact
内容逻辑：自媒体是门户网站的前置触点，核心任务是「让陌生人认识团队、产生兴趣，引流至门户完成深度信任建立」。

本 Skill 只负责内容撰写阶段，输出供人工审阅和多轮修改的纯文字稿件。内容确认后，调用 media-formatter Skill 完成排版。

---

## 执行流程 (Workflow)

### Step 1：信息确认

如用户未提供以下信息，主动询问：
- 选题 / 主题方向
- 目标发布平台：公众号 / 小红书 / LinkedIn / 多平台组合
- 有无原始素材或参考附件
- 目标受众（设计师 / 产品经理 / 出海客户 / 决策者）
- 有无禁止添加的数据或观点

### Step 2：判断栏目归属

对照下方「栏目体系」，明确本次内容属于哪个栏目，调取对应写作规范。

### Step 3：生成标题（必须输出 3-5 个）

读取 `references/case-library.md` 中对应栏目的案例后再生成标题。

标题写作黄金法则：
- 制造认知缺口：读者看完产生「我不知道这个，我要看」的冲动
- 痛点精准命中：直接说出目标受众的真实困惑或渴望
- 数字增强信任：「3个」「7步」「90%的人」等具体化表达
- 对比制造张力：「vs」「却」「但」「反而」等转折词
- 禁止 AI 腔：不用「探索」「深度剖析」「全面解读」「赋能」「打造」「构建」等套话

各栏目标题风格：
- 《AI设计实战》→ 操作感强，有工具名，有结果预期。例：「用 Figma AI 做完整套设计系统，我省了多少时间？」
- 《设计思考/增长视角》→ 有观点，有反常识。例：「为什么你的设计越来越「对」，客户却越来越不买单？」
- 《出海洞察》→ 直指决策者痛点。例：「花了 30 万做出海官网，这 5 个地方正在悄悄流失客户」
- 《行业点评》→ 有态度，有争议性。例：「XX 品牌改版了，我不得不说几句实话」
- 《事务所日常》→ 真实感，不端着。例：「我们设计团队内部评审是什么样的？」

### Step 4：撰写正文

根据平台分别撰写（规范见「各栏目写作规范」章节）。支持多轮对话修改，每次修改后重新输出完整稿件。

## 栏目体系

共五个栏目，每次出稿前必须先判断选题归属哪个栏目，再调用对应写作规范。

栏目一：《设计思考》《增长视角》
- 内容定位：证明能力，引流门户，拉开与普通设计号的差距
- 主发平台：公众号、LinkedIn
- 字数要求：1800-2500字
- 发布频率：每周1篇

栏目二：《AI设计实战》
- 内容定位：获取流量，强化团队AI能力标签
- 主发平台：小红书、即刻、掘金
- 字数要求：公众号版 1200-1800字；小红书版 300-600字
- 发布频率：每周1-2篇

栏目三：《出海洞察》
- 内容定位：精准触达出海客户决策者，主打商业转化
- 主发平台：公众号、LinkedIn
- 字数要求：1500-2000字
- 发布频率：每周1篇

栏目四：《行业点评》
- 内容定位：扩大曝光，引发互动，借热点建立观点型人设
- 主发平台：小红书、即刻
- 字数要求：500-800字
- 发布频率：每月1篇

栏目五：《事务所日常》
- 内容定位：雇主品牌建设，展示团队温度，服务招募目标
- 主发平台：小红书、即刻
- 字数要求：300-500字
- 发布频率：每季度1篇

## 各栏目写作规范

### 《设计思考》《增长视角》（公众号深度文）

内容方向：出海官网设计复盘、B端产品改版案例、设计系统决策思路

结构模板：
1. 引子：用真实场景或反常识观点开场，制造共鸣或困惑（1-2段）
2. 核心观点：提炼1个核心洞察，用小标题拆解3-5个支撑点
3. 真实案例：展开项目经验，呈现「挑战→思考→方案→成果」完整链路
4. 方法论提炼：给读者可复用的思维框架或操作路径
5. 结尾CTA：引导至门户深度阅读 / 商务咨询入口

写作要求：
- 像一个有经验的设计师在和你聊项目，不是写论文
- 案例必须具体，有细节，有决策过程，不能只讲结论
- 禁止用「首先、其次、最后」做段落开头，改用更自然的过渡
- 每个大段建议配图，标注 [配图建议：XXX]

---

### 《AI设计实战》（小红书 / 公众号）

内容方向：AI工具测评、AI辅助设计工作流、Figma AI实战、AI图在品牌设计中的边界

公众号版结构：
1. 钩子开头：直接说结果，如「用这个方法，我把出海官网首屏的设计时间压缩到了2小时」
2. 背景/场景：描述使用场景，让读者对号入座
3. 工具/方法拆解：分步骤详细讲，操作截图处标注 [配图：操作截图]
4. 真实效果对比：前后对比或量化结果
5. 踩坑提示：1-2个真实注意事项，增加可信度
6. 结尾CTA：私信获取资料包 / 关注获取更多

小红书版结构：
1. 开头1-2句直接说价值，不废话
2. 短段落，每段2-3行
3. 用 ⚡️✅🔧💡 等 emoji 作视觉分割
4. 步骤类用 1️⃣2️⃣3️⃣ 编号
5. 结尾提问引导评论
6. 附5-8个话题标签

写作要求：
- 用「我」「我们」「试了一下」「发现」等第一人称，加入主观感受
- 不得编造工具功能或未经验证的效果数据
- 工具名保持准确：Figma、Claude、Gemini、扣子、NotebookLM

---

### 《出海洞察》（公众号 / LinkedIn）

内容方向：出海官网避坑、不同市场视觉偏好、DTC品牌体验诊断

结构模板：
1. 痛点开场：用出海客户最常见的误区或困境开场
2. 问题分析：拆解深层原因（市场差异/文化差异/设计逻辑）
3. 解决框架：可操作的改进方向 + 具体案例
4. 市场洞察：目标市场的视觉/体验/转化偏好
5. 行动建议：明确下一步 + 咨询入口

写作要求：
- 受众是出海决策者（CEO/CMO/品牌总监），少讲设计美学，多讲转化和商业结果
- 结尾必须有CTA：「欢迎后台留言，获取出海官网设计诊断服务」

---

### 《行业点评》（小红书 / 即刻）

内容方向：大厂品牌改版点评、出海品牌视觉拆解、官网问题诊断

结构模板：
1. 观点前置：直接亮出立场，有态度
2. 案例拆解：分析改版/设计的3-5个维度（视觉/交互/品牌/转化）
3. 有一说一：指出优点，也直接说不足，保持独立判断
4. 行业延伸：从案例延伸到行业共性问题
5. 引发讨论：结尾留开放性问题

写作要求：
- 观点鲜明，不和稀泥，但基于事实，不造谣
- 配图必须有改版前后对比截图，标注 [配图：改版前后对比]

---

### 《事务所日常》（小红书 / 即刻）

内容方向：设计师工作日常、内部评审片段、新人入职故事、团队工具清单

结构模板：
1. 场景还原：像讲故事一样描述真实内部场景
2. 细节展现：工具名、对话片段、决策瞬间
3. 团队文化点：自然带出团队价值观，不刻意
4. 人才吸引CTA：「如果你也喜欢这样的工作方式，我们在招人」

写作要求：
- 追求「真实感」和「温度」，不追求专业感
- 可以有小瑕疵、有玩笑、有内部黑话（适度）

## 撰写规范与限制 (Constraints)

所有栏目通用，每次输出前必须自检：

1. 去AI感：禁止「探索」「深度剖析」「赋能」「全面」「助力」「打造」「构建」「融合」
2. 不造数据：所有数据、案例必须来自用户提供的真实素材，不得凭空捏造
3. 保留原意：附件/参考内容的主要观点必须全部呈现，不得曲解或遗漏
4. 人感表达：多用第一人称、口语化过渡、真实感细节
5. CTA必须有：每篇文章结尾必须有明确的行动引导
6. 配图必标注：需要配图的位置用 [配图建议：XXX] 标注，描述要具体

```
- 爆款标题 1（好奇心型）：...

- 爆款标题 3（行业高度型）：...

### 3. 正文内容（公众号版，纯文字，供审阅与修改）
[完整文章，含 [配图建议：XXX] 标注]

### 4. 小红书版本（如需）
[独立的小红书版本，含话题标签]

### 5. LinkedIn 版本（如需）
[英文原创，200-400词，针对出海/B端决策者，突出商业价值与设计专业度，结尾附1个行动引导]

### 6. 配图清单
[汇总所有配图位置和建议内容]
```
