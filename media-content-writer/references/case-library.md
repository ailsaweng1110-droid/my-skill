## 使用说明

每次出稿时，根据当前稿件的栏目类型，优先读取对应栏目的案例。重点学习：
- 开头方式：用哪种钩子？情绪型还是信息差型？
- 文章结构：几个段落？怎么推进？
- 爆款原因：这篇为什么火？用相同逻辑套用到当前选题

### 案例 AI-01

- 标题：谷歌Stitch「氛围设计」干崩Figma 8.8%股价：十年经验，败给巨头一次更新（附实测）
- 来源：机器之心 | 微信公众号 | 2026-03-21
- 数据：阅读 30000 / 点赞 421 / 分享 3737（分享极高，强传播型）

**开头方式：**
情绪炸点式。用极端表达「Figma，危矣！」开场，同步绑定权威对象（Google/Figma）和资本数据（股价跌8.8%），三秒内制造「紧张+好奇」的复合情绪。

**文章结构（7段推进）：**
1. 情绪炸点开场 → 制造危机感
2. 新概念解释（Vibe Design）→ 降低理解门槛，用类比拆解
3. 产品能力展示 → 不讲原理，只讲「能做到什么」
4. 实测教程（核心爆点）→ 手把手操作 + Prompt示例 + 全链路闭环，让读者从「看新闻」变成「想去试」
5. 社会反馈引用 → 支持派/反对派/中立派，制造认知冲突，提升讨论欲
6. 行业上升 → 从产品升级到时代趋势，拔高格局
7. 焦虑给出口式收尾 → 「与其焦虑，不如去用」，引导行动，防止「爽完就走」

**爆款核心逻辑：**
- 标题三连爆点：巨头 + 受害者 + 情绪（焦虑/替代），技术革命+职业危机模型
- 强情绪贯穿：「设计师完了」「十年经验没用」制造职业代入感，天然易传播
- 信息+情绪+观点三位一体，内容不枯燥
- 可操作性极强（Prompt示例+操作路径），实用性=分享动机
- 不给结论，让读者站队，裂变核心

### 案例 AI-02

- 标题：Google Stitch + Claude Code，DESIGN.md让Vibe Design更有设计感
- 来源：硅基鹿鸣 | 微信公众号 | 2026-03-21
- 数据：阅读 8397 / 点赞 144 / 分享 1326

**开头方式：**
信息差型轻量钩子。不靠情绪恐吓，用高密度信息（更新+新概念+免费+竞品股价）吸引理性读者，比情绪型更容易在专业圈传播。

**文章结构（10段推进）：**
1. 轻量信息差开场
2. 作者背书（「我之前测过…」）→ 建立「真实测评」可信度
3. 实验设计：同一Prompt测四种方案 → 把「体验」升级成「实验」
4. Stitch测试结论（优缺点同讲）→ 增强可信度
5. Claude Code测试 → 打压预期，为反转铺垫
6. 转折点：发现DESIGN.md（隐藏能力被挖出）→ 全篇爆点核心
7. 方法验证与闭环
8. 三元素抽象总结（Stitch=设计/Claude=工程/DESIGN.md=桥）→ 帮读者记住一个模型
9. 可复制操作指南 → 降低行动门槛
10. 个人IP收口

**爆款核心逻辑：**
- 输出「方法」而非「信息」，工作流型内容传播力强于评测型
- 强信息差：DESIGN.md是大多数人忽略的隐藏能力，爆款=发现别人忽略的东西
- 实验对比而非主观感受，「像论文一样严谨，但读起来不累」
- 可执行性比有观点更容易传播
- 无过度焦虑，更容易被理性用户转发进专业圈
