# media-content-writer · 自媒体内容撰写 Skill

> 负责内容撰写阶段，从选题到定稿。排版请使用 → [`media-formatter`](../media-formatter/README.md)

---

## 安装方式

**前置条件**：已在龙虾 Agent 平台创建或进入目标 Agent。

**第一步：上传文件**
将以下文件按原有目录结构上传至 Agent 的 Skill 管理区：

```
media-content-writer/
├── SKILL.md
└── references/
    └── case-library.md
```

**第二步：配置 Skill**
在 Agent 平台的 Skill 设置中，将 `SKILL.md` 指定为该 Skill 的主入口文件，`references/` 文件夹设为关联资源目录。

**第三步：验证触发**
在对话框输入以下任意一句话，确认 Skill 正常触发：

```
帮我写一篇关于 AI 设计工具的公众号文章
```

Agent 应主动询问选题方向、平台、受众等信息，说明 Skill 已生效。

> 具体操作路径因平台版本而异，如界面有差异请参考龙虾官方文档。

---

## 这个 Skill 是做什么的

给定一个选题或素材，输出可供人工审阅、支持多轮修改的自媒体文章草稿，覆盖公众号正文、小红书版本、LinkedIn 英文版本。

**不包含**：HTML 排版、公众号发布格式。内容定稿后交给 `media-formatter` 处理。

---

## 文件结构

```
media-content-writer/
├── SKILL.md                      ← Agent 执行指令主文件
├── README.md                     ← 本文档（供人阅读）
└── references/
    └── case-library.md           ← 行业爆款参考案例库（11条，按栏目分类）
```

## 如何使用

### 基础用法

直接告诉 Agent 你的选题方向，Skill 会自动触发：

```
「帮我写一篇关于 Figma AI 新功能的文章，发公众号」
「我想出一篇出海官网避坑的稿子，目标客户是做跨境电商的」
「小红书写一篇 AI 生图打光技巧的笔记」
```

### 带素材用法（推荐）

提供原始素材可以显著提升内容质量：

```
「基于这份项目复盘文档，帮我写一篇《设计思考》栏目的文章」
「参考附件里的 AI 工具测评记录，出一篇公众号稿」
```

### 多轮修改用法

内容撰写支持多轮对话打磨，告诉 Agent 哪里需要调整即可：

```
「第二段太学术了，改得口语化一点」
「结尾 CTA 力度不够，重新写」
「整体字数压缩到 1500 字以内」
```

## 每次输出包含

```
1. 选题分析        ← 栏目归属 + 受众痛点判断
2. 标题矩阵        ← 3个不同类型的爆款标题方案
3. 公众号正文      ← 纯文字版，含配图建议标注
4. 小红书版本      ← 如需，独立版本含话题标签
5. LinkedIn版本    ← 如需，英文原创200-400词
6. 配图清单        ← 汇总所有配图位置和建议内容
```

---

## 内容定稿后

Agent 会提示：「内容已完成，可调用 **media-formatter** Skill 进行公众号排版。」

将定稿正文和选定标题交给 `media-formatter`，即可获得可直接发布的 HTML 版本。

---

## 案例库维护

`references/case-library.md` 收录了11条行业高互动案例，Agent 出稿时会自动参考对应栏目的爆款逻辑。

**更新方式**：从飞书多维表格导出最新 CSV，交给维护人员更新 `case-library.md` 并追加新条目即可，无需改动 SKILL.md。

建议每月更新一次，保持案例库与行业热点同步。

---

## 注意事项

- **不要让 Agent 编造数据**：所有数据和案例必须来自你提供的真实素材
- **素材越具体，内容越好**：提供真实项目背景、截图说明、客户反馈等细节，输出质量会显著提升
- **标题多选几个再决定**：Skill 默认输出3种类型标题，建议结合发布时间和热点选择，也可以让 Agent 继续生成更多方案
